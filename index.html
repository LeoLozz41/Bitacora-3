Frameworks

Los frameworks se utillizan en el ámbito de la programación de aplicaciones desde hace décadas.
Recientemente se han usado para programar y diseñar aplicaciones web, por lo que ya existen decenas de frameworks para CSS.
Un frameworks es un conjunto de herramineta, librerías, convenciones, y buenas prácticas que pretenden encapsular las tareas repetitivas en módulos genéreicos fácilmente reutilizables.
Un frameworkses un conjunto de herramientas, hojas de estilos y buenas prácticas que permiten al diseñador web olvidarse de las tareas repetitivas para centrarse en los ellemetnos únicos de cada diseño en los que pueden aportar valor.

Unidad resposivas

Las unidades absolutas
Las unidades absolutas, como px(pixeles), cm (centímetros) e in (pulgadas), representan tamaños fijos y no cambian independientemente del contexto en ele que se utilicen, proporcionado una medida exacta que pueden ser útil para diseños que requieren precisión milimentrica.
Unidades relativas
Las uinidades relativas como % (porcentajes), em, rem, son felxibles y se adaptan en función de otros elementos.
Por ejemplo, % es relativo al tamaño del contenedor padre, em es relativo al tamaño de fuente del elemento padre, y rem es relativop al tamaño de fuiente del elemento raíz.
Las unidades erelativas son ideales para crear diseños resposivos, ya que permiten que los elementos del diseño cambien en respuesta al tamaño de la pantalla.
Unidades de vista de ventana
Las unidades de vista de ventana son herramientas que se ajustan dinámicamente al tamaño de la ventana gráfica del nevegador. Las unidades de vista de ventana son herramientas que se ajustan dinámicamente al tamaño de la ventana gráfica del navegador.
Las unidades de vista de ventana son: vw, vh, vmin, vmax.

Mobile first

Una investigacion realizada por la GSMA afirma que el uso de internet a través de dispositivos móviles crecerá un 50% hasta 2020.
Estar conectado en internet se volvió parte de nuestro día. Poco a poco estamos dejando de usar la computadora para realizar las treas cotidinas y estamos usando cada vez más los smartphones.
En 2015, Google ya hablaba sobre esta tenedcia, y priorizó los sotios web que son mobile friendly en los resultados de las búsquedas.
Los dispositivos móviles garantizam la practicidad que buscamos pero, para que la experiencia en internet sea buena, es necesario que la navegación sea pensada para estos dispositivos. Si las personas pasan más tiempo navegando een internet por medio del teléfono móvil; lo mas sensato seria crar primero un sitio web optimizado para los dispositivos móviles.
el modelo Mobile First es una filossofia que perfecciona el proceso de diseño de cualquier sítio web, buscando mejorar el diseño web adaptable.
Para aplicar este método , debemos tener en cuenta que todo lo que no sea esencial debe quedar fuera de nuestro diseño.

Modulo grid

Comúnmente de han utilizado varias estrategias de CSS para diseñar los páginas web. Primero, se usó table, luego float, position y display, pero todos estos métodos omiten funciones importantes.
Grid es un sistema de diseño bidmencional en cuadrículas que, en comparacion con cualquier sistema de diseño web del pasado, cambia por complejo la forma en que diseñamos las interfaces de usasurio.
Grid fue creado especificaamente para resolve rlos problemas de diseño. Adiferencia de Flexbox cuyo flujo unidireccional tiene diferentes casos de uso, Grid permite definir una esquema de cuadrícula.
Desde marzo de la mayoria de los navegadores ofrecán soporte nativo sin prefijos para CSS Grid: Chrome, Firefox, Safari, Opera, Internet Explorer 10 y 11, con el prefijo.
display: grid;
Para comnezar, se dene definir un elemento contenedor como una cuadricula con: display:grid; Todos los hijos directo del contenedor de la cuadricula se cinvierte automáticamente en elementos del grid.

@supports

El objetivo de este tema es que los estudiantes comprendan el uso de la regla @supports en CSS para detectar la compatibilidad de características CSS en diferentes navegadores y apliquen estilos condicionales basados en esta compatibilidad para asegurar una experiencia de usuario consistente.
Mejora progresiva
La mejora progresiva es una estrategia de diseño web que consiste en crear una versión básica de la funcionalidad para todos los navegadores, y luego agregar características mejoradas para navegadores que las soporten.
Degradación elegante.
Por otro lado, se refiere a la estrategia de crear la versión más avanzada de una función y asegurarse de que se degrade de manera aceptable en navegadores más antiguos o manos capaces.
